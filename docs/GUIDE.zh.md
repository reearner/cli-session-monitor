# 使用指南

逐步介绍这个小部件能做的一切。第一次用?从 **快速上手** 开始,再按需查阅对应章节。

> English: [GUIDE.md](GUIDE.md)
>
> 语言:界面默认跟随系统语言(English / 中文)自动切换,也可在 **设置 → 语言** 手动指定。

---

## 快速上手

1. **启动** 应用。它会显示为一个小面板(并在系统托盘有一个图标)。
2. 打开 **设置**(⚙ 按钮)→ **接入** → 在 *Claude Code* 一行点击 **安装接入**。
   这是唯一需要做的设置。(Codex 无需安装,见下文。)
3. 像平常一样使用 Claude Code / Codex。每个会话会显示为一张**卡片**,带状态和实时计时;
   每当一轮结束你会收到桌面通知。

> 只想先看看效果?用 `CSM_DEMO=1` 启动,会进入自带演示(假数据,不读盘、不联网)。

---

## 接入你的 CLI

### Claude Code(一键,可还原)

**设置 → 接入 → Claude Code → 安装接入。** 这会往 `~/.claude/settings.json` 追加一个小小的
上报 *hook*。它是:

- **只追加** —— 你已有的设置和 hook 都会保留。
- **先备份** —— 改动前先写一份带时间戳的副本。
- **可还原** —— **卸载** 只移除本应用添加的条目。

接入生效后,该行显示 **已接入 ✓**。

### Codex(自动 —— 无需安装)

Codex 通过**只读**方式监视:跟踪它的会话文件(`~/.codex/sessions/**/rollout-*.jsonl`)。
不改配置、不用安装。该行显示 **自动监视 ✓**。(如果旧版本本工具曾写入过 `notify` 条目,
点 **卸载旧 notify** 即可清理。)

---

## 看懂卡片

每个会话一张卡片:

- **状态圆点 + 文字** —— `运行中`、`等待输入` 或 `空闲`。一轮结束会显示为*等待输入*
  (轮到你回复了)。
- **实时计时** —— 运行时往上走;结束后显示上一轮耗时。Codex 的计时带 **`估算`** 标记,
  因为它的计时是近似的。
- **`▸` 启动目录** 和 **`🪟` 匹配到的编辑器窗口**(或*未找到对应窗口*)。
- **高亮** —— 你**当前所在**的那个编辑器窗口对应的卡片会被高亮,一眼就能看出在看哪个会话。
- **关闭(×)** —— 隐藏你不关心的卡片。该会话再次活动时它会自动回来。

## 跳转到编辑器(点击卡片)

**点击卡片** 即可把它对应的 **Cursor / VS Code** 窗口切到前台 —— 包括通过 **Remote-SSH**
打开的窗口(见下文)。应用会记住某个目录用的是哪个编辑器,并重新打开那一个。

### 优化跳转(仅当两个文件夹同名时需要)

匹配靠的是**窗口标题里显示的文件夹**。如果你同时开着两个**同名**文件夹的窗口
(比如两个不同的 `app` 目录),仅凭标题无法区分。

一次性解决:**设置 → 编辑器跳转 → 优化跳转。** 这会往 VS Code / Cursor 写入一个
`window.title` 设置,让标题带上**完整路径**。它会先备份、不会覆盖你已自定义的标题,
点 **还原** 可撤销。此后跳转和"当前窗口高亮"都会精确命中。

> 限制:跳转聚焦的是一个**窗口**,不是某个具体的终端标签页。同一个编辑器窗口里的多个 CLI
> 都对应这一个窗口。

---

## 悬浮球与边缘停靠

打开 **设置 → 轻量模式**。然后:

- **点击标题栏** 把面板收成一个**悬浮球**。
- 悬浮球会**停靠到最近的屏幕边缘**,变成一条细条,显示计数(**▶** 运行中 / **!** 待处理)。
- **悬停**细条会弹回小球;**点击**重新打开完整面板;**拖动**到任意位置可重新停靠到新边缘。
- 有完成 / 等待事件时它会**闪烁**提醒你。
- 它的位置**重启后会记住**。找不到了?托盘 → **重置悬浮球位置**。

## 远程监控(另一台机器上的会话)

你可以监视跑在**远端服务器**上的 CLI 会话,而不需要你自己开 SSH 隧道。两端的连接靠一个
**[ntfy](https://ntfy.sh) 主题**:桌面应用**订阅**一个主题;远端的一个小**代理(agent)**
**发布**到**同一个**主题。

1. **设置 → 远程。** 设一个**不易猜到的主题**(或点 *导出* 自动生成一个)。可选填自建中继
   地址和访问令牌。
2. 点 **导出远端脚本** → 生成 `remote-agent.sh`(写到 `~/.cli-session-monitor/`),主题 /
   地址 / 令牌都**烘焙在里面**。
3. 把两个小二进制放到远端、和 `remote-agent.sh` 同目录。最省事:从
   [最新 release](https://github.com/reearner/cli-session-monitor/releases/latest)
   下载 **`csm-remote-x86_64-linux.tar.gz`** 解压(musl 全静态,**远端无需 Rust**,任意
   x86_64 Linux 可跑):
   ```bash
   tar -xzf csm-remote-x86_64-linux.tar.gz   # 得到 ./csm-agent 和 ./session-reporter
   ```
   (其他方式:用 `CSM_AGENT_BIN` / `CSM_REPORTER_BIN` 指向已有路径;或在本仓库检出目录里
   装好 Rust 让它 `cargo build`。)
4. 在远端运行:`bash remote-agent.sh`。
5. 回到桌面(已订阅该主题),远端的会话就会出现,并标注远端**主机名**。(刚设好主题?
   重启一下应用让它去订阅。)

想在一个视图里看到多台远端,就在每台上跑**同一个主题**——桌面只显示它所订阅的那一个主题。
导出的脚本在不同主题之间**只有 `CSM_RELAY_TOPIC=` 这一行不同**。

**它监控什么。** agent 两种 CLI 都管:**Codex 自动**纳入(读 rollout 文件);**Claude Code**
需要在那台远端装上报 hooks。一步装好:

```bash
bash remote-agent.sh --install-claude   # 在本机装 Claude hooks,然后运行
```

直接 `bash remote-agent.sh` 则只跑 agent、不碰 Claude(只有 Codex)。

**要一直开着——它是常驻中继,不是一次性命令。** `csm-agent` 会前台常驻、盯着会话活动并
实时发布事件;一旦退出,桌面就收不到更新了。所以让它一直开着,或放后台:

```bash
# nohup 后台(日志写文件)
nohup bash remote-agent.sh > ~/csm-agent.log 2>&1 &
tail -f ~/csm-agent.log        # 看到 “publishing …” 两行即 OK;Ctrl-C 退出 tail 不影响后台

# 或放进可重连的 tmux 会话
tmux new -s csm 'bash remote-agent.sh'     # 脱离:Ctrl-b d  ·  回看:tmux attach -t csm
```

**一个 agent 同时管两种 CLI——别重复开。** 单个 `csm-agent` 进程会同时 tail Codex 的 rollout
文件**并**读 Claude 的事件总线,两路都发到同一个 topic。不需要开第二个。查一下开了几个:

```bash
pgrep -af csm-agent      # 正常应只有一行
```

多开只会把同样的事件发两遍(桌面按会话去重,无害但浪费)——`pkill -f csm-agent` 后只起一个即可。

**如何停止 / 卸载。** agent 不装任何持久化东西——**Ctrl-C**(或 `pkill -f csm-agent`)即停。要移除它装的 Claude hooks:

```bash
bash remote-agent.sh --uninstall        # 移除 Claude hooks(会先备份)
```

之后删掉 `remote-agent.sh` 即可。本地则在 **设置 → 远程** 把主题清空即停止订阅。

> **按用户安装,非系统级。** 一切都作用于**运行命令的那个用户**——只动该用户的 `~/.claude`、
> 读其 `~/.codex`,不涉及全局。远端要监视哪个用户的 CLI,就**以该用户身份**运行(用该用户 SSH
> 登录,或 `sudo -u 用户 bash remote-agent.sh --install-claude`);要看多个用户就每个用户各跑一份。

> **隐私:** 公共 `ntfy.sh` 主题任何知道它的人都能读。请用不易猜到的主题,和/或自建 ntfy +
> 令牌。只发布元数据(来源、主机、目录、状态、时间戳)——**绝不**包含对话内容。

### "访问令牌"是什么?

它是**可选**的——只有当你把主题锁起来、不让所有人都能读时才需要。ntfy 把它当作
**Bearer 凭据**,在*发布*(agent)和*订阅*(本应用)两端都会带上;设置后,只有持有令牌的人
才能读取或发布你的会话元数据。用开放的公共主题时留空即可。要保护一个主题:

- **ntfy.sh:** 注册账号,**预留(reserve)**该主题并设为拒绝匿名访问,然后生成一个
  **访问令牌**(Account → Access tokens),粘贴到这里。
- **自建 ntfy:** 开启鉴权,创建一个用户 + 令牌,并给该主题配置可读写的 ACL。

## 在 Remote-SSH 下工作

这就是常见的那种情况:*"我用 VS Code/Cursor 的 Remote-SSH 扩展编辑远端文件夹,并在那个窗口
的终端里跑 Claude/Codex。"* 这里其实有两件事在发生,分清楚会更好理解:

- **会话跑在远端**(它的终端在服务器上)。要**看到它的状态**,需要按上面的 **远程监控**
  配置好——在那台服务器上运行 `csm-agent`。这样卡片就会显示该会话,并带上远端主机名。
- **编辑器窗口跑在本机**(Remote-SSH 只是代理文件;窗口本身在你的 Windows 桌面上)。所以
  **点击卡片照样能聚焦那个窗口**——跳转和当前窗口高亮和本地会话完全一样。

所以一套完整的 Remote-SSH 用法是:**服务器上跑 `csm-agent`(用于状态)** + **点击卡片聚焦
你本机的 Remote-SSH 窗口(用于跳转)**。如果跳转不太准,跑一次 **优化跳转**(见上)——它对
Remote-SSH 的标题同样有效。

> 如果你*只*想聚焦窗口、不需要实时状态,其实连 agent 都不用——但那样卡片要等中继上报后才会
> 出现。

## 系统托盘

右键托盘图标:

- **显示 / 隐藏** —— 切换面板显隐。
- **重置悬浮球位置** —— 当小部件跑到屏幕外时把它找回来。
- **退出** —— 完全退出。(直接关窗口只是收进托盘。)

## 常见问题排查

- **卡片显示"未找到对应窗口"。** 该目录的编辑器没开,或两个文件夹同名——打开它,或运行
  **优化跳转**。
- **高亮错了 / 没高亮。** 同样的原因;**优化跳转** 能让匹配精确。空闲会话从不高亮。
- **没有通知。** 检查 **设置 → 完成时桌面通知**,以及 Windows 里本应用的通知设置。
- **首次启动 SmartScreen 警告。** 当前发布版还没做代码签名——*更多信息 → 仍要运行*。
  (已在 `docs/POLISH.md` 记录。)
- **需要诊断信息?** 用 `CSM_DEBUG=1` 启动,它会写 `~/.cli-session-monitor/csm.log`。
  默认关闭。
